# CI/CD & Source Control

## Purpose

Version, test, and promote changes safely and repeatably across environments.

## Outcomes

Branching strategy, automated deployments, and rollback/hotfix procedures.

## Decisions to Make

- Repo strategy & branching model
- Deployment method (Deployment Pipelines/APIs/YAML)
- Config-as-code and environment variables
- Release approvals and rollback

## Standards & Patterns

- Pull request reviews and checks
- Automated tests in the pipeline
- Semantic versioning of releases

## Deliverables & Artifacts

- Repo structure & branch policies
- Release pipeline definitions
- Promotion gates & artifact list

## Checklist

- [ ] Repos created & linked
- [ ] Branch policies enforced
- [ ] CI tests passing
- [ ] Rollback tested
